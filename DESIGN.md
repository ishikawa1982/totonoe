# ととのえ 設計書

> [!ポイント]
> 本書は Markdown 自動整形ツール **「ととのえ」** の設計書です。この文書自体がサンプルを兼ねており、**見出し・表・コードブロック・コールアウト・チェックリスト**など多様な記法で、自動整形の実力を一望できます。

最終更新：2026-06-26 ／ 区分：*内部設計資料*

---

## 1. 概要

「ととのえ」は、Markdown を貼り付けて **「整える」** を押すだけで、内容に応じた配色・フォント・レイアウトへ自動整形する _単一ファイルの Web ツール_ です。社内ドキュメントを「ぱっと見て分かる」状態にすることを目的としています。

- **入力**：Markdown テキスト（貼り付け／`.md` ファイルのドラッグ＆ドロップ）
- **出力**：画面表示・印刷用 PDF・スタイル同梱の HTML
- **動作**：すべてブラウザ内で完結し、本文は外部送信しません

### 1.1 設計方針

1. 専門知識ゼロでも「整える」一発で読みやすくする
2. 自動結果を起点に、テーマ・配色・フォントを手で微調整できる
3. 依存を最小化し、配布は 1 ファイルのみ

## 3. 機能仕様

### 3.1 自動判定ロジック

入力された Markdown を走査し、次の指標からテーマを推定します。

| 指標 | 判定への影響 |
| --- | --- |
| コードブロック数 | 多い → **技術文書** |
| 表の行数 | 多い → **報告書** |
| 見出し数・本文量 | 少見出し＋長文 → **記事・説明** |

判定の擬似コードは次のとおりです。

```javascript
function heuristicSpec(md) {
  const codeBlocks = countFences(md);   // コードブロック数
  const tableRows  = countTableRows(md); // 表の行数
  const headings   = countHeadings(md);  // 見出し数

  if (codeBlocks >= 2) return theme("standard", "技術文書");
  if (tableRows  >= 6) return theme("report",   "報告書");
  if (headings   <= 3) return theme("article",  "記事・説明");
  return theme("standard", "技術文書");
}
```

### 3.2 テーマ一覧

各テーマは見出しとレイアウトを大きく切り替えます。

- **技術文書** … 標準。下線＋左バーの見出し
- **報告書** … 枠付きヘッダー＋全幅の罫線
- **提案書** … 塗りつぶしの帯見出し（インパクト重視）
- **マニュアル** … 手順マーカー（▶）付き・教科書体
- **仕様書** … 細い罫線で囲む技術調
- **記事・説明** … 中央寄せのエディトリアル
- **折りたたみ** … 見出しクリックでセクションを開閉

### 3.3 対応 Markdown 記法

> [!補足]
> この設計書は、対応記法のショーケースも兼ねています。下のチェックリストの項目は、すべて本書内で使用済みです。

- [x] 見出し（h1〜h4）
- [x] **太字**・*斜体*・~~打ち消し線~~
- [x] 箇条書き／番号付きリスト（入れ子可）
- [x] 表（GFM）
- [x] コードブロック（言語別ハイライト）
- [x] コールアウト（注意／ポイント／補足）
- [x] 引用ブロック
- [x] リンクと `インラインコード`
- [x] 水平線
- [ ] 画像（任意・本書では未使用）

入れ子リストの例：

- フォント
  - ゴシック（既定）
  - 明朝
  - 丸ゴシック／UDゴシック／教科書体
- 配色
  1. 和色プリセット 12 色
  2. 任意色（カラーピッカー）

#### 3.3.1 コードブロックの例

スタイル定義（CSS）:

```css
.doc h2 {
  border-left: 5px solid var(--accent);
  padding-left: 0.6em;
}
```

整形設定の内部表現（JSON）:

```json
{
  "theme": "report",
  "font": "gothic",
  "accent": "#284B6F",
  "showToc": true
}
```

起動コマンド（PowerShell）:

```powershell
Start-Process ".\totonoe.html"
```

## 4. 画面構成

左に入力、右にプレビューを置く 2 ペイン構成です。

```
+----------------+----------------------------+
|  入力ペイン     |  プレビューペイン           |
|  ・テキスト入力 |  ・整形レポート（テーマ等） |
|  ・整えるボタン |  ・整形後ドキュメント       |
|  ・微調整       |  ・PDF / HTML 書き出し      |
+----------------+----------------------------+
```

### 4.1 微調整パネル

整形後に、**テーマ・配色・フォント・目次**を手動で上書きできます。

> 設定の変更は再解析なしで即時に反映されます。プレビューを見ながら最終調整できます。

---

## 5. 処理フロー

1. Markdown を入力する
2. **「整える」** を押す（または `Ctrl` ／ `⌘` + `Enter`）
3. 内容を解析し、テーマ・配色・フォント・目次を自動決定
4. プレビューへ反映
5. 必要に応じて微調整し、**PDF** または **HTML** で書き出す
